Confidential ◆ tier

Hardware-attested inference for the rare operator who has the silicon for it. This is a gated, data-center-only tier - not a self-serve toggle.

What it is

A confidential node proves, with hardware attestation, that inference runs inside a Trusted Execution Environment whose memory the host cannot read. The broker verifies a real AMD SEV-SNP quote (signature chain, freshness/binding, and an allow-listed launch measurement) before granting the ◆ badge, and confidential demand routes only to verified nodes.

Why it needs data-center hardware

Real confidentiality has to protect the data where it is processed. For LLM serving that means two TEEs, not one:

  • CPU TEE - AMD SEV-SNP (EPYC Milan / 7003 or newer). Encrypts VM memory and produces the signed attestation quote.
  • Confidential GPU - NVIDIA H100-class (Hopper+) in CC mode. The model and KV-cache live in VRAM, which SEV-SNP does not protect - so a CPU TEE alone is not enough.

Both are data-center parts. There is currently no consumer path to attested confidential GPU inference - not AMD (Threadripper / Ryzen lack SEV-SNP), not Intel (SGX deprecated on consumer, TDX is Xeon-only), and no consumer GPU has a confidential mode. Anyone advertising "confidential" on a gaming GPU is overstating it.

Don't have the hardware? You still earn.

The standard tier is the path for everyone else, and it pays the same way. RogerAI's consumer-grade guarantee is not "the host can't see your data" - it is verifiable, attributable serving: every request carries a co-signed lineage receipt (signed by the serving node, counter-signed by the broker, hash-chained), so which node served which request can be proven and cannot be forged. Just run roger share.
