How to run a local LLM on your own GPU

You already own the compute. Here is the plain-English way to turn your GPU into a private, free AI endpoint - what you need, which models fit, and how to plug it into the apps you already use.

Quick answer. To run a local LLM on your own GPU, install a runtime (Ollama, llama.cpp, or LM Studio), download an open model like gpt-oss, Qwen, or Llama, and point any app at its OpenAI-compatible endpoint. A 12-16 GB GPU comfortably runs a strong 7-14B model; for larger models, or to skip setup, tune in to a community GPU for free.

Running a large language model on your own machine means no subscription, no per-token meter, and no prompts leaving your control. The tooling got easy in 2026: if you have a gaming GPU, you can be chatting with an open model in a few minutes.

§1 · The basics

What do you need to run a local LLM?

Three things: a GPU with enough VRAM (video memory is the real limit, not the chip speed), a runtime that loads the model and speaks the OpenAI API, and an open-weights model file. That is it - no cloud account, no key.

  • GPU / VRAM. 8 GB runs a 7-8B model; 12-16 GB runs 13-14B; 24 GB runs ~32B or gpt-oss-20b; 48 GB+ runs a 70B or gpt-oss-120b quantized.
  • Runtime. Ollama (one-command, easiest), LM Studio (a friendly app), or llama.cpp (max control). All expose an OpenAI-compatible /v1/chat/completions.
  • Model. Open weights you download once: gpt-oss, Qwen, Llama, Gemma, Mistral. Quantized (Q4/Q5) versions shrink VRAM with little quality loss.
§2 · The free path

How do you run a local LLM for free?

Every piece above is free and open source. The fastest route with Ollama:

  • Install the runtime, then pull a model: ollama run qwen2.5:7b
  • It downloads once and drops you into a chat - fully offline after that.
  • Want a bigger brain? ollama run gpt-oss:20b if your VRAM allows.

The only cost is the electricity your GPU already draws. There is no token meter and no monthly bill.

§3 · Sizing

Which models can your GPU actually run?

Match the model's parameter count (and quantization) to your VRAM. As a rule of thumb for a Q4 quant: budget roughly 0.6 GB of VRAM per billion parameters, plus a couple of GB for context. A 24 GB card handles a 32B model or a fast 20B; a 12 GB card is happiest around 13-14B. See what real operators are actually serving on the RogerAI model directory for live examples and prices.

§4 · Wiring it up

How do you use your local model from other apps?

Because the runtime speaks the OpenAI API, almost any tool works. Point it at your local endpoint (usually http://localhost:11434/v1 for Ollama) with any placeholder key. Coding assistants like aider, chat UIs like open-webui, and your own code (the OpenAI SDK with a changed base_url) all just work - no code changes beyond the URL.

§5 · When yours isn't enough

What if your GPU is too small - or you have one to spare?

Two common cases, one tool. If your GPU can't fit the model you want, you can tune in to someone else's: roger use gpt-oss-120b gives you a local OpenAI-compatible proxy backed by a community GPU, with free models on the band right now. And if you have idle GPU, you can put it on the air with roger share - free for the community, or priced to earn. It is the same OpenAI endpoint, just backed by real hardware you or a neighbor owns. The full walkthrough is in the operating manual.

§6 · FAQ

Frequently asked questions

Is running a local LLM really free?
Yes. The runtimes and open models are free and open source; you only pay for the electricity your GPU already uses. There is no subscription or per-token charge.
What GPU do I need for a local LLM?
Any GPU with 8 GB+ of VRAM runs a capable 7-8B model. 12-16 GB runs 13-14B models comfortably; 24 GB runs ~32B or a fast 20B; 48 GB+ handles 70B-class or gpt-oss-120b quantized.
Can I run a local LLM without a GPU?
You can run small models on CPU, but it is slow. A better option is to keep the same OpenAI-compatible workflow and tune in to a community GPU - free models are available on RogerAI.
How do I connect a local LLM to my apps?
Point the app at your runtime's OpenAI-compatible URL (e.g. http://localhost:11434/v1) with any placeholder API key. Tools like aider, open-webui, and the OpenAI SDK need only that base-URL change.
What is the best free local LLM right now?
It depends on your VRAM: Qwen and Llama variants at 7-14B are excellent all-rounders, and gpt-oss (20B / 120B) is strong if you have the memory. Check the model directory for what people are actually running.
How do I run a bigger model than my GPU allows?
Use a smaller quantization, or route the request to a bigger GPU without leaving your workflow: roger use <model> proxies to a community node over the same OpenAI API, so your app is unchanged.
