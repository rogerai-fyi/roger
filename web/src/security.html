Security

How the platform is built to fail safe, and how to report a problem.

Our posture

Nodes reach the broker over an outbound-only tunnel - an operator's machine opens the connection out, so there are no inbound ports to expose on a home network. Work is pulled, never pushed in.

Every settled request carries an Ed25519-signed lineage receipt, so which node served which request can be verified and cannot be forged.

The browser talks to the broker over credentialed CORS against an allow-listed origin; the session lives in a signed, http-only cookie and no access tokens ever touch page JavaScript.

Stripe holds all PCI card data and all KYC identity documents. We never store card numbers, bank details, or identity documents on our side.

Report a vulnerability

Please report security issues privately - do not open a public issue or PR.

Preferred: GitHub private vulnerability reporting on the repo (Security → Report a vulnerability). Or email security@rogerai.fyi.

We aim to acknowledge within 72 hours. Please include reproduction steps and impact; we will coordinate a fix and a disclosure timeline with you. Full policy: SECURITY.md. Machine-readable contact: /.well-known/security.txt (RFC 9116).

Report abuse or a station

For abuse, prohibited content, or a misbehaving station - not a software vulnerability - use the report control on each station in the Models directory, or email abuse@rogerai.fyi. The Acceptable Use Policy is in the Terms. We act on reports, and a station's record follows it.
